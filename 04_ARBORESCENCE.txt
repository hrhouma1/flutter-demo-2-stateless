|                       RESUME POUR LES ETUDIANTS                       |

Ce projet montre comment:

[OK] Organiser un projet Flutter en plusieurs fichiers
[OK] Separer les pages et les widgets reutilisables
[OK] Utiliser les imports relatifs (../ et ./)
[OK] Creer des widgets Stateless avec parametres
[OK] Composer des widgets simples pour creer des interfaces complexes
[OK] Documenter son code pour faciliter la comprehension
[OK] Suivre les bonnes pratiques Flutter

Points cles a retenir:

1. Un fichier = une responsabilite
2. pages/ = ecrans complets
3. widgets/ = composants reutilisables
4. Tout widget Stateless commence par: class MonWidget extends StatelessWidget
5. Les parametres sont definis avec 'final' et 'required'
6. La methode build() definit ce que le widget affiche
